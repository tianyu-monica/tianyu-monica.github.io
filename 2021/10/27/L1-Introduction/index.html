L1 Introduction

Posted on 2021-10-27 In Static Analysis Views:

背景：过去十年中，语言的内核几乎没有大的改变，但程序越趋复杂

挑战：面对大规模的复杂程序，如何确保其可靠安全？

静态分析的作用：

  • 程序可靠性
    • 空指针
    • 内存泄漏
  • 程序安全性
    • 私人信息泄露
    • 注入攻击
  • 编译优化
    • 死代码删除
  • 程序理解
    • IDE

定义：Static analysis analyzes a program P to reason about its behaviors and determines whether it satisfies some properties before running P.

然而，莱斯定理指出我们并无法获得一个准确完美的答案。

Rice‘s Theorem:

Any non-trivial property of the behavior of program in a r.e. language is undecidable.

于是，退而求其次，寻求尽可能有用的方案：

通过Overapproximate或者Underapproximate以获得Sound或Complete的结果。但会分别出现False Positives与False Negatives。

静态分析就是在确保soundness的基础上，在分析精度和速度上做一个平衡。

两个词简要概括静态分析（针对大多数的静态分析，即may analysis）

  • Abstraction
  • Over-approximation
